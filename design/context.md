# Context

## Goal

将现有 Excel 年度学习计划迁移为线上 Web 系统，使技术架构与开发专业线成员可以完成能力自评、年度计划制定、执行跟踪和成果验收，同时让 Buddy 提供指导，让 Leader 掌握团队能力差距与学习进展。

## Users and permissions

- 成员：维护本人当前周期的自评、计划、进展和成果。
- Buddy：查看绑定成员的数据，审批年度计划，评论指导，验收或退回学习成果。
- Leader：查看全团队数据；兼任管理员，维护账号、成员—Buddy 关系、能力模型、学习材料和学习周期。
- 每名成员对应一名 Buddy；绑定关系保留历史。

## Core jobs

- 成员在 30 分钟内完成自评并提交可执行的年度学习计划。
- Buddy 清晰处理待审批、待验收和退回补充任务。
- Leader 快速识别延期、能力差距、重点能力覆盖和团队提升趋势。

## Confirmed workflow

1. 成员查看全部能力项并完成自评。
2. 系统根据入选能力项生成年度计划，成员可以调整月份、任务、验收方式和预计耗时。
3. Buddy 审批计划；退回时成员修改后再次提交。
4. 成员执行计划，追加多条进展记录并提交文字、链接和附件证据。
5. Buddy 验收通过后计划项完成；退回时成员补充后再次提交。
6. Leader 通过团队看板和管理中心管理全流程。

## Constraints

- V1 使用系统内置账号密码，Leader 创建和维护账号。
- 预计 20 人以内、低并发、仅桌面浏览器。
- 技术栈无强制要求，采用 Django 单体应用、PostgreSQL 和 Docker Compose。
- 部署到现有公有云服务器；目前只有服务器，没有域名和 HTTPS 证书。
- 邮件仅实现接口与发送日志，不真实发送。
- 基础数据仅从 Excel 模板导入；成员计划在线重新填写。
- 支持个人和团队汇总 Excel 导出。
- 学习周期支持自然年和从指定日期开始的连续 12 个月。
- 旧周期归档后彻底只读。

## Success criteria

- 成员可在 30 分钟内完成自评并提交年度计划。
- Buddy 能从统一待办中完成计划审批、成果验收和退回。
- Leader 能从看板下钻到成员和计划项，定位延期、能力差距和提升趋势。
- 核心权限、状态流转、归档、导入导出和文件访问测试全部通过。
- Docker 生产构建、数据库迁移和备份恢复演练通过。

## Scope V1

- 本地账号、角色权限和成员—Buddy 绑定。
- 能力模型、学习材料和学习周期维护。
- 能力自评、计划生成、Buddy 审批。
- 多条进展、评论、成果提交与 Buddy 验收。
- 个人、Buddy 和 Leader 工作台。
- 站内待办、邮件模拟日志、文件上传。
- Excel 基础数据导入、个人计划和团队汇总导出。
- 历史周期只读归档和关键数据审计。
- Docker Compose 部署到公有云服务器。

## Non-goals V1

- 企业 SSO。
- 手机端适配。
- 真实邮件发送。
- AI 自动推荐学习计划。
- 在线课程播放。
- 绩效考核和薪酬关联。

## Assumptions

- 现有 Excel 模板是能力项和学习材料初始化的权威来源。
- 公有云服务器可提供 Docker、持久化磁盘和必要网络端口。
- 正式录入真实账号和学习数据前会补充域名和 HTTPS。

## Risks

- 310 个能力项可能让自评超过 30 分钟，需要折叠分组、键盘操作、批量填写、自动保存和进度提示。
- 能力模型变更同步到未完成计划时可能覆盖成员修改，必须展示差异并由 Leader 明确选择同步范围。
- 服务器本地附件需要纳入备份和恢复流程。
- 无 HTTPS 时只能用于验收，不可承载真实账号和学习数据。
