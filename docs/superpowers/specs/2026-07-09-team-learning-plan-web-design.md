# 团队年度学习计划 Web 系统设计

日期：2026-07-09

## 1. 背景与目标

现有 Excel 模板以能力模型为基础，包含能力差距自评、年度学习计划、执行跟踪、月度复盘、学习材料和个人看板。Web 系统应保留这条数据链路，同时解决多人协作、Buddy 指导、审批验收、历史归档和团队分析问题。

V1 面向技术架构与开发专业线，预计 20 人以内、低并发、桌面浏览器使用。

## 2. 角色和权限

### 成员

- 查看全部启用能力项和学习材料。
- 编辑本人当前周期的自评和草稿计划。
- 提交计划给 Buddy 审批。
- 记录进展，提交成果文字、链接和附件。
- 查看 Buddy 评论、审批和验收结果。
- 查看本人历史归档周期。

### Buddy

- 查看当前绑定成员的数据。
- 审批或退回成员年度计划。
- 查看进展并发表评论。
- 验收或退回学习成果。
- 查看历史指导关系对应的只读数据。

### Leader

- 查看团队看板和所有成员明细。
- 创建、停用和重置账号。
- 设置角色并维护成员—Buddy 绑定。
- 维护能力模型、学习材料和学习周期。
- 选择是否将基础数据修改同步到未完成计划。
- 执行周期归档和 Excel 导出。

服务端必须对每个查询和写操作校验角色、对象归属、绑定关系、周期状态和业务状态。界面隐藏按钮不能替代服务端权限。

### 公共导航

- 消息待办
- 历史周期

### Leader 管理中心

## 4. 设计方向

采用“角色工作台”作为整体组织方式：

- 成员围绕“自评→计划→执行→验收”推进。
- Buddy 围绕“待审批→待验收→退回跟踪”工作。
- Leader 围绕“团队指标→风险→下钻→管理”工作。

自评页使用高密度数据表，提高 310 个能力项的处理效率；个人工作台使用少量年度时间线和里程碑，表达成长过程。

## 5. 核心业务流程

### 5.1 年度计划

```text
草稿自评
  → 生成计划草稿
  → 提交 Buddy
  → 待审批
      ├─ 退回 → 恢复编辑 → 再次提交
      └─ 通过 → 执行中
```

### 5.2 成果验收

```text
执行中
  → 成员提交成果
  → 待验收
      ├─ 退回 → 补充执行 → 再次提交
      └─ 通过 → 已完成
```

退回必须填写原因。状态变更通过 django-fsm-2 定义，不允许直接赋值绕过转换。

### 5.3 周期归档

- 周期可以按自然年或指定开始日期连续 12 个月。
- 归档前展示待审批、待验收和未完成任务检查结果。
- Leader 明确确认后归档。
- 归档后所有业务对象只读，任何角色都不能重新开启或修改。

## 6. 核心页面

### 6.1 成员工作台

- 当前周期、计划状态和下一步主操作。
- 自评完成度、入选计划项数、本月任务。
- 临期、延期和退回补充提醒。
- 最近进展和 Buddy 评论。
- 年度时间线与月度完成趋势。

### 6.2 能力自评

- 按能力类别、一级能力域和二级能力项折叠。
- 支持按职级、差距、优先级、是否入选和填写状态筛选。
- 固定能力名称列，直接编辑当前/目标掌握度、优先级、季度和月份。
- 自动计算差距和推荐行动。
- 支持键盘连续填写、批量设置计划时间和批量纳入计划。
- 修改自动保存；保存失败保留本地输入并允许重试。
- 显示已评估数量和计划入选数量。

### 6.3 年度计划

- 从入选能力项一键生成。
- 保存能力编码、名称、层级、材料、任务、验收方式和预计耗时快照。
- 按季度和月份分组。
- 成员可调整月份、任务、验收方式和预计耗时。
- 提交后锁定；Buddy 退回后恢复编辑并显示意见。

### 6.4 执行跟踪

- 每个计划项展示目标、材料、截止时间和状态。
- 成员追加多条进展记录和实际耗时。
- 时间线展示进展、评论、成果提交和验收记录。
- 成果支持文字、外部链接和多个受限附件。

### 6.5 Buddy 工作台

- 待审批计划。
- 待验收成果。
- 退回后待补充项。
- 被指导成员进度、临期和延期风险。
- 最近指导评论。

### 6.6 Leader 工作台

- 指标：参与人数、完成率、延期、待审批和待验收。
- 图表：能力差距分布、能力域覆盖和月度完成趋势。
- 风险：延期、长期无进展和反复退回。
- 按周期、成员、Buddy、能力域和状态筛选并下钻。

### 6.7 管理中心

#### 能力模型

- 以“能力类别→一级能力域→二级能力项→三级能力项”树形维护。
- 新增、编辑、排序、启用和停用。
- 维护编码、名称、建议职级、预计耗时、验收方式和推荐行动。
- 已被引用的能力项不可物理删除。
- 保存前展示关联材料、当前计划和未完成计划影响范围。
- 保存后选择仅影响新计划，或同步到指定未完成计划。
- 同步前展示字段差异；不覆盖计划中的成员自定义字段。

#### 学习材料

- 维护编码、名称、类型、来源、说明、链接/附件和状态。
- 与能力项建立多对多关系。
- 按能力域、类型和状态筛选。
- 停用后不再进入新计划，历史快照保持不变。

#### 用户与指导关系

- 创建、停用和重置账号密码。
- 设置成员、Buddy 和 Leader 角色。
- 维护成员—Buddy 当前绑定和历史关系。
- Buddy 仍有待审批或待验收任务时，必须先转交才能停用或换绑。

#### 学习周期

- 创建自然年或连续 12 个月周期。
- 设置参与成员、开始和结束时间。
- 执行归档检查并确认归档。

## 7. 数据模型

### 身份

- User：Django 自定义用户模型。
- Mentorship：成员、Buddy、生效和失效时间。

### 基础数据

- CapabilityCategory：能力类别。
- CapabilityDomain：一级/二级能力层级。
- CapabilityItem：三级能力项、编码、职级、耗时、验收方式、推荐行动、排序和启用状态。
- LearningMaterial：编码、名称、类型、来源、说明、链接/附件和状态。
- CapabilityMaterial：能力项与材料的多对多关系。

### 周期和自评

- LearningCycle：类型、开始、结束和状态。
- CycleParticipant：周期与成员。
- Assessment：成员、周期、能力项、当前/目标掌握度、差距、优先级、是否入选和计划时间。

### 计划和执行

- LearningPlan：成员、周期、Buddy、审批状态和审批记录。
- PlanItem：能力快照、材料快照、月份、任务、验收方式、耗时和执行状态。
- ProgressUpdate：计划项、作者、内容、耗时和时间。
- GuidanceComment：计划或计划项、作者、内容和时间。
- EvidenceSubmission：计划项、提交说明、链接、提交批次和验收状态。
- EvidenceAttachment：成果提交附件。
- ReviewDecision：审批或验收决定、意见、操作者和时间。

关键模型接入 django-simple-history。业务时间线来自 ProgressUpdate、GuidanceComment、EvidenceSubmission 和 ReviewDecision，不直接读取审计表生成。

## 8. 基础数据和快照规则

- Excel 模板中的 `02_能力差距自评` 初始化能力层级和三级能力项。
- `06_学习材料索引` 初始化学习材料。
- 能力项中的材料编码解析为多对多关系；未知编码导致整批导入失败。
- 导入先完成字段、编码唯一性、层级关系、材料关系和行数预检，再在单个数据库事务中写入。
- 计划生成时保存显示和验收所需的完整快照。
- Leader 选择同步时，仅更新指定未完成计划的来源字段。
- 成员已经调整的月份、任务、验收方式和预计耗时默认不覆盖；若 Leader 要覆盖，界面必须逐字段明确选择。
- 已完成项和归档周期永不同步。

## 9. 看板指标

- 参与人数：当前周期参与成员数。
- 计划完成率：已完成计划项数 / 有效计划项总数。
- 延期项：已超过截止时间且未完成的计划项。
- 待审批：处于待审批状态的年度计划。
- 待验收：最新成果提交处于待验收状态的计划项。
- 能力差距：目标掌握度减当前掌握度，不小于零。
- 提升趋势：按月统计验收通过项对应能力差距的变化。
- 长期无进展：达到可配置的固定天数阈值且未完成；V1 默认 30 天，配置位于代码常量。
- 反复退回：同一计划项累计退回两次及以上。

所有汇总可下钻到构成该数字的成员和计划项。

## 10. Excel 导入导出

### 初始化导入

- 输入：现有 V1.3 Excel 模板。
- 结果：能力层级、三级能力项、材料和关联关系。
- 不导入任何成员自评、计划或进度。

### 个人导出

- 当前或历史周期基本信息。
- 自评结果。
- 年度计划。
- 进展、成果和验收结论。

### 团队导出

- 成员和 Buddy。
- 计划、完成、延期和耗时统计。
- 能力差距和能力域覆盖明细。

导出使用 openpyxl 生成固定格式工作簿，并检查公式错误、字段数量和样例数据。

## 11. 异常与并发

- 自动保存失败时不清空界面输入，并提供明确重试入口。
- 状态机拒绝重复或非法提交、审批和验收。
- 写操作携带版本号；版本不一致时提示刷新并比较最新数据。
- 关键状态转换在数据库事务中完成。
- 文件上传限制扩展名、MIME、大小和数量；文件名由服务端生成。
- 下载附件前重新校验对象权限。
- Excel 导入失败时回滚整批事务。
- 所有越权请求返回一致错误，不泄露对象是否存在。

V1 默认附件限制：

- 单文件不超过 20 MB。
- 每次成果提交最多 5 个附件。
- 允许 PDF、常见办公文档、图片、ZIP 和纯文本。

## 12. 技术架构

- Backend/UI：Django 单体应用和服务端模板。
- Database：PostgreSQL。
- UI kit：Tabler，按需引入。
- Tables：django-tables2。
- Filters：django-filter。
- Workflow：django-fsm-2。
- Audit：django-simple-history。
- Import/export：admin 场景使用 django-import-export；业务 Excel 使用 openpyxl。
- Charts：Chart.js。
- Deployment：Docker Compose，包含 Web、PostgreSQL 和反向代理。
- Media：服务器持久化卷；数据库和附件一起备份。
- Email：Django 文件邮件后端，保留生成内容和发送日志，不连接外部服务。

不引入 REST API、前端 SPA、Celery、Redis、对象存储、动态工作流引擎或 BI 系统。

## 14. 测试和验收

### 权限和流程

1. 成员只能编辑本人当前周期。
2. Buddy 只能审批、评论和验收绑定成员。
3. Leader 可管理团队和基础数据。
4. 计划审批、成果退回和再次提交状态正确。
5. 归档后任何写操作失败。
6. 非法和重复状态转换失败。

### 基础数据

7. Excel 导入得到 310 个三级能力项，材料及关联关系正确。
8. 能力层级、编码唯一性及排序正确。
9. 已使用能力项只能停用，不能删除。
10. 能力同步只影响选定的未完成计划。
11. 停用材料不进入新计划，历史快照保留。
12. Buddy 停用或换绑前必须完成任务转交。

### 报表和文件

13. 个人和团队 Excel 导出字段及内容正确。
14. 看板汇总与下钻明细一致。
15. 文件上传限制有效，授权用户可下载，越权访问失败。

### 上线门槛

- 自动化测试通过。
- 生产镜像构建通过。
- 数据库迁移在空库和已有测试数据上通过。
- 数据库和附件备份恢复演练通过。
- 成员、Buddy、Leader 三条端到端验收流程通过。
- 未配置 HTTPS 时仅允许测试数据验收；真实数据上线必须配置域名和 HTTPS。
